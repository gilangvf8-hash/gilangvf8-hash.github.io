    <section>
      <h2>Apa yang terjadi pada MyWapBlog?</h2>
      <p>MyWapBlog adalah layanan blogging mobile yang populer di kalangan pengguna ponsel; pemilik mengumumkan penutupan layanan dan periode migrasi pada akhir 2016 sehingga banyak pengguna diminta mem-backup dan memindahkan konten mereka sebelum tanggal penutupan. 0</p>
    </section>

    <section>
      <h2>Mengapa orang mencari pengganti?</h2>
      <ul>
        <li>Ketersediaan konten: ketika platform tutup, pemilik konten perlu tempat baru untuk menyimpan arsip blog.</li>
        <li>Kontrol dan kepemilikan: ingin kontrol penuh atas file, domain, dan tata letak tanpa bergantung pada layanan gratis pihak ketiga.</li>
        <li>Biaya & portabilitas: solusi yang murah/atau gratis tapi mudah dimigrasi dan dapat dihosting sendiri atau di layanan lain.</li>
      </ul>
    </section>

    <section>
      <h2>Apakah GitHub (Pages + Jekyll) cocok sebagai pengganti?</h2>
      <p>Singkatnya: <strong>Ya — untuk banyak kasus</strong>. GitHub Pages memungkinkan Anda meng-host situs statis (termasuk blog) gratis dari repositori GitHub; biasanya dipasangkan dengan generator statis seperti Jekyll untuk mengubah file Markdown menjadi halaman HTML. Ini menjadikannya pilihan yang populer bagi pembuat blog yang menginginkan hosting gratis, kontrol versi, dan dukungan custom domain. 1</p>

      <div class="note">
        Catatan penting: GitHub Pages adalah host untuk situs statis (HTML/CSS/JS). Jika Anda membutuhkan CMS ber-interface grafis (editor WYSIWYG, manajemen komentar bawaan tanpa konfigurasi), GitHub sendiri bukanlah CMS penuh—tetapi kombinasi GitHub + layanan pihak ketiga (mis. Netlify CMS, Staticman, Disqus/Comento untuk komentar) dapat menutup celah tersebut. 2
      </div>
    </section>

    <section>
      <h2>Perbandingan singkat: MyWapBlog vs GitHub Pages</h2>
      <table>
        <thead>
          <tr><th>Aspek</th><th>MyWapBlog (lama)</th><th>GitHub Pages + Jekyll</th></tr>
        </thead>
        <tbody>
          <tr><td>Kemudahan pembuatan</td><td>Sangat mudah, UI berbasis web untuk pengguna ponsel</td><td>Perlu sedikit teknis (git, repositori, Markdown/Jekyll) — banyak template mempermudah.</td></tr>
          <tr><td>Biaya</td><td>Gratis</td><td>Gratis untuk hosting dasar (publik) di GitHub Pages</td></tr>
          <tr><td>Kontrol & backup</td><td>Terbatas pada fungsi platform</td><td>Penuh — semua file ada di repository Git (mudah backup/migrasi)</td></tr>
          <tr><td>Fitur interaktif</td><td>Biasa ada komentar/guestbook bawaan</td><td>Butuh integrasi pihak ketiga untuk komentar, formulir, dll.</td></tr>
          <tr><td>Portabilitas</td><td>Sulit saat platform tutup</td><td>Mudah — Anda punya repo dan bisa memindahkannya atau meng-host di mana saja</td></tr>
        </tbody>
      </table>
    </section>

    <section>
      <h2>Langkah dasar migrasi dari MyWapBlog ke GitHub Pages</h2>
      <ol>
        <li><strong>Backup konten lama:</strong> Ekspor atau salin posting Anda dari MyWapBlog (teks, gambar, tanggal). Lakukan ini terlebih dulu saat masih bisa diakses. 3</li>
        <li><strong>Buat akun GitHub & repository:</strong> Daftar di GitHub lalu buat repositori baru dengan nama <code>username.github.io</code> untuk publikasi langsung (opsional: repositori proyek + branch <code>gh-pages</code>). 4</li>
        <li><strong>Pilih generator statis / tema:</strong> Jekyll adalah pilihan umum (bawaan GitHub Pages mendukung Jekyll). Pilih tema yang mendukung layout blog (Minima, etc.) atau gunakan starter template. 5</li>
        <li><strong>Konversi posting ke Markdown:</strong> Buat file <code>_posts/YYYY-MM-DD-judul.md</code> untuk setiap posting (format Jekyll) atau halaman Markdown biasa. Sertakan metadata front matter (judul, tanggal, tags). 6</li>
        <li><strong>Upload gambar & aset:</strong> Simpan folder <code>assets/</code> atau <code>images/</code> dalam repo; perbarui path pada postingan.</li>
        <li><strong>Setup custom domain (opsional):</strong> GitHub Pages mendukung domain kustom dengan pengaturan DNS yang sesuai. 7</li>
        <li><strong>Tambahkan komentar / form:</strong> Pakai layanan seperti Disqus, Staticman, atau form/publish service lain bila perlu.</li>
        <li><strong>Publish & cek:</strong> Commit dan push ke GitHub; GitHub Pages akan membangun dan menayangkan situs Anda (periksa build logs bila pakai Jekyll). 8</li>
      </ol>
    </section>

    <section>
      <h2>Kelebihan & Kekurangan (ringkas)</h2>
      <h3>Kelebihan</h3>
      <ul>
        <li>Gratis untuk hosting statis publik dan dukungan custom domain. 9</li>
        <li>Kontrol penuh atas file, mudah backup dan portabilitas lewat Git.</li>
        <li>Banyak tema, tutorial, dan komunitas yang mendukung migrasi. 10</li>
      </ul>

      <h3>Kekurangan</h3>
      <ul>
        <li>Kurva belajar untuk pengguna non-teknis (git, Markdown, Jekyll). 11</li>
        <li>Fitur dinamis (komentar, login pengguna, posting lewat UI) memerlukan integrasi tambahan.</li>
      </ul>
    </section>

    <section>
      <h2>Kesimpulan</h2>
      <p>GitHub Pages (dengan Jekyll atau generator statis lain) adalah alternatif yang solid dan modern untuk menyimpan & menerbitkan arsip blog yang semula ada di MyWapBlog, terutama bila Anda mengutamakan kontrol, portabilitas, dan biaya rendah. Namun apabila Anda perlu editor visual bawaan atau fitur komunitas yang langsung tersedia tanpa konfigurasi teknis, Anda mungkin perlu menambahkan layanan pihak ketiga atau memilih CMS lain (mis. WordPress, Blogger) tergantung kebutuhan.</p>
    </section>
